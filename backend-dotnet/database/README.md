# Smart Factory AI Dashboard Database

The backend runs on **SQL Server** (local `.\SQLEXPRESS`, database `SmartFactory`,
Windows Authentication) using **EF Core code-first**. The C# entities in
`backend-dotnet/Models/Database/` are the single source of truth for the schema.

## Files in this folder

1. `seed.sql`: demo/development data. Executed once on startup by `DatabaseSeeder`
   when the database has no rows yet. Still the source of the seed dataset.
2. `schema.sql`: the original hand-written schema. **Reference only** — it is no
   longer executed. The live schema is generated from the EF Core entities.

## How the schema and data are created

- **Schema:** generated by EF Core migrations (in `backend-dotnet/Migrations/`).
  On startup the app calls `Database.Migrate()`, which creates the database and
  applies any pending migrations. snake_case table/column names are produced by
  `EFCore.NamingConventions` so they match the original design.
- **Data:** `DatabaseSeeder.Seed(...)` runs `seed.sql` when the database is empty
  (idempotent — it does nothing once seeded).

## Working with the model (code-first workflow)

```bash
# after changing an entity in Models/Database/
dotnet dotnet-ef migrations add <MigrationName>
dotnet dotnet-ef database update
```

The connection string lives in `backend-dotnet/appsettings.json` under
`ConnectionStrings:SmartFactorySqlServer`. Verify connectivity at runtime with
`GET /health/database`.

## Main data areas

1. Users and roles / permissions.
2. Factory areas.
3. Production lines and records.
4. Machines and machine issues.
5. Warehouse zones, items, IO/BU masters, and movements.
6. Safety alerts and camera events.
7. Employees, skills, and shift plans.
8. Form requests and approval steps.
9. Notifications.
10. AI recommendations.
11. Report snapshots.
