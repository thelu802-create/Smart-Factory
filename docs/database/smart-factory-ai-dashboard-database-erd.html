Data flow

How the web reads the database

The current C# MVC backend reads SQLite first through SampleDataService. If SQLite is unavailable, it falls back to sample-data.json. Later, the same controllers can move SQL reads into repositories.

Database TablesProduction, warehouse, safety, workforce, forms, notifications, reports.
SampleDataServiceSQLite-first loading, DB auto-seed, and JSON fallback for demo safety.
ServicesBusiness mapping, calculated KPIs, recommendation summaries.
MVC ControllersFeature API groups such as WarehouseController and DashboardController.
React FrontendDashboard pages fetch data from localhost:8000.

Implementation notes

What matters when extending the DB

These notes help keep the MVC backend, SQLite tables, frontend tables, and future repositories aligned.

Warehouse search

  • Search by io_id, io_code, bu.
  • Also search by item code, item name, and batch.
  • Keep item_code unique for fast lookup.

Dashboard KPIs

  • Output and completion come from production_lines.
  • Safety alert count comes from safety_alerts.
  • Warehouse signals come from warehouse_items.status.

Logical references

  • notifications.related_entity_id is flexible by design.
  • It can point to safety, production, forms, or warehouse records.
  • Add stricter foreign keys later if the domain becomes stable.
