Project Summary

Smart Factory AI Dashboard is a web-based platform for simulating an intelligent factory. It helps users monitor production, manage warehouse operations, detect safety risks, analyze productivity, plan workforce allocation, automate internal forms, and generate reports.

The system does not need real AI models at the beginning. AI behavior is simulated through rule-based recommendations and mock analytics — the focus is a realistic dashboard experience on simulated data.

Recommended Technology Stack

Frontend

1React
2Vite
3TypeScript
4React Router
5CSS Modules / plain CSS initially
6Recharts or Chart.js

React suits the many reusable UI parts — KPI cards, tables, alerts, forms, filters, charts, and dashboard panels.

Backend

1C#
2ASP.NET Core MVC / Web API
3Microsoft.Data.Sqlite
4SQLite (MVP) → PostgreSQL (full)

C# / ASP.NET Core fits the Visual Studio workflow, strongly typed API contracts, and MVC organization. SQLite gives the MVP a real local database without a separate server.

User Roles

Five roles define access and permissions. Role-based access should be planned from the beginning; each role sees only the modules and actions it is allowed.

Factory Manager

Full oversight & approvals
  • View all dashboard data & production lines
  • View & resolve safety alerts
  • Approve shift plans, overtime & forms
  • View, export reports & review AI recommendations

Production Manager

Line monitoring & issues
  • View production dashboard & line details
  • Create / review machine issue reports
  • Review productivity analytics
  • Receive alerts & request workforce support

Warehouse Manager

Inventory & storage
  • View warehouse status, search & track items
  • Review AI storage suggestions
  • Approve import/export requests
  • Resolve wrong placement alerts & view reports

Safety Officer

Risk & incident response
  • View AI camera alerts & factory risk map
  • Record safety incidents
  • Resolve or escalate safety alerts
  • View safety reports

Employee

Self-service & reporting
  • View assigned shift and work line
  • Submit leave request forms
  • Submit overtime request forms
  • Submit machine issue reports & receive notifications

Business Rules

Production

  • Warning when efficiency below 85%
  • Stopped when downtime exceeds threshold
  • High defect rate creates a production alert
  • Low output triggers AI root-cause analysis

Warehouse

  • Occupancy above 90% → capacity warning
  • Wrong zone → wrong-placement alert
  • Quantity below minimum → low inventory
  • AI storage weighs category, frequency, capacity

Safety

  • Restricted-zone entry → at least High severity
  • Critical alerts appear on dashboard immediately
  • Every alert ends: resolved / escalated / false / investigation

Workforce & Forms

  • Available < required → staffing shortage alert
  • Overtime only when target unreachable
  • AI shift recs reviewable/editable before applying
  • Approved import/export updates movement records

UI & UX Requirements

Layout

  • Dashboard layout with sidebar + topbar
  • Sidebar contains main modules
  • Topbar: search, notification count, profile
  • Important alerts visible without scrolling (desktop)
  • Cards, tables, charts easy to scan

Visual — Status Colors

Normal / SuccessGreen
WarningAmber
Critical / DangerRed
Default infoNeutral

Operational, modern, professional — not a marketing landing page. Dense information, remaining readable.

Responsive

  • Desktop: sidebar + multi-column panels
  • Tablet: reduced dashboard columns
  • Mobile: sections stack vertically
  • Text must not overflow cards or buttons

Interaction & Accessibility

  • Cards clickable when they drill down
  • Alerts have clear action buttons
  • Forms show validation errors clearly
  • Loading, empty & error states per module
  • Keyboard accessible; color not the only severity cue

MVP Scope & Out of Scope

✓ In MVP

  • Dashboard with mock KPIs & alerts
  • Sidebar navigation
  • Production overview + line detail
  • Warehouse overview + item search
  • Safety alert list + detail
  • Workforce planning + AI suggestion
  • Forms submit/approval + Reports preview
  • Notification center

✕ Out of Scope

  • Real camera integration
  • Real AI model training
  • Real-time sensor connection
  • Complex authentication
  • PDF / Excel export implementation
  • Advanced role-permission editor
  • Multi-factory support
  • Live WebSocket updates
The MVP runs with the C# backend and SQLite sample database. The frontend keeps local mock data as a fallback when the backend is not running.

Validation & System States

Validation Rules

  • Required fields not empty; valid dates
  • Numeric fields never negative
  • Efficiency & defect rate between 0–100
  • Item code unique; batch not empty for tracked items
  • Severity ∈ Low, Medium, High, Critical
  • Resolve needs note for High/Critical alerts
  • Rejection reason required when rejecting

Required States (per module)

Loading"Generating optimized schedule"
Empty"No item found for this code"
Error"Unable to load factory summary. Retry"
SuccessAction completed successfully
Denied"You do not have permission…"
